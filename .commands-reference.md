## 🚀 Docker Compose (Development) — WORKING ✅

### Start Stack
```bash
make up
# Equivalent: docker compose -f deploy/compose/docker-compose.yml --project-directory . up -d --build
```

### Check Status
```bash
make ps
# or: docker compose -f deploy/compose/docker-compose.yml --project-directory . ps
```

### View Logs

**All services:**
```bash
docker compose -f deploy/compose/docker-compose.yml --project-directory . logs -f
```

**Specific service:**
```bash
make logs S=pwa              # Frontend
make logs S=api-service      # Backend API
make logs S=edge-service     # Edge processing
make logs S=mosquitto        # MQTT broker
make logs S=influxdb         # Time-series DB
make logs S=grafana          # Dashboards
make logs S=sensor-simulator # Sensor emulator
make logs S=telegraf         # Ingestion
```

### Stop Stack
```bash
make down
# or: docker compose -f deploy/compose/docker-compose.yml --project-directory . down
```

### Restart Services
```bash
docker compose -f deploy/compose/docker-compose.yml --project-directory . restart pwa
```

### Test API Endpoints
```bash
curl http://localhost:8000/health
curl http://localhost:8000/api/parcels
curl http://localhost:8000/api/recommend/zoneA
curl http://localhost:8000/api/predict/yield/zoneA
```

### Run Smoke Test
```bash
make smoke
```

### Scale Services
```bash
make scale N=3  # Scale sensor-simulator to 3 replicas
```

### Run Load Test
```bash
make loadtest
```

### Run Unit Tests
```bash
make test-api  # API tests (pytest)
make test-pwa  # PWA tests (Vitest)
```

## ☸️ Kubernetes (Orchestration) — Network Issue ⚠️

### Current Status
- **Issue**: Docker Hub image pull fails (`connection reset by peer` on `kindest/node`)
- **Root Cause**: Network connectivity issue with Docker Hub registry from your environment
- **Workaround**: Use Docker Compose (above) for now

### If You Need Kubernetes Later

**Check if kind cluster exists:**
```bash
kind get clusters
```

**Create cluster (requires working Docker Hub access):**
```bash
make k8s-cluster
# or manual: kind create cluster --config deploy/k8s/kind-config.yaml
```

**Deploy to Kubernetes (after cluster is created):**
```bash
make k8s-up
```

**Check status:**
```bash
make k8s-status
```

**View pod logs:**
```bash
make k8s-logs S=api-service
make k8s-logs S=edge-service
make k8s-logs S=grafana
```

## 🔧 Useful Diagnostic Commands

### View Container Resources
```bash

```

### View Docker Network
```bash
docker network inspect agri-iot-compose-k8s-cloud_frontend
docker network inspect agri-iot-compose-k8s-cloud_backend
```

### Inspect a Container
```bash
docker inspect agri-iot-compose-k8s-cloud-api | head -50
```

### Shell into Container
```bash
docker exec -it agri-iot-compose-k8s-cloud-api /bin/bash
docker exec -it agri-iot-compose-k8s-cloud-pwa /bin/sh
```

### View Image Details
```bash
docker image ls | grep agri-iot
docker image inspect agri-iot-compose-k8s-cloud/api-service:latest
```

## 🛠️ Configuration

Edit `.env` to customize:
```bash
cp .env.example .env
# Edit .env, then: make down && make up
```

Key variables:
- `MQTT_USERNAME` / `MQTT_PASSWORD` — MQTT broker credentials
- `INFLUX_TOKEN` — InfluxDB admin token
- `GRAFANA_PASSWORD` — Grafana admin password
- `API_AUTH_PASSWORD` — API authentication
- `JWT_SECRET` — JWT signing key

## 🧹 Clean Up

**Stop without removing volumes:**
```bash
make down
```

**Stop and remove everything (data is deleted):**
```bash
make clean
# or: docker compose -f deploy/compose/docker-compose.yml --project-directory . down -v
```

**Remove Kubernetes cluster (if created):**
```bash
make k8s-delete          # Remove resources
make k8s-cluster-delete  # Remove entire cluster
```
